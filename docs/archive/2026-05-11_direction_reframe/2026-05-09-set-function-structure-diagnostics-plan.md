## Diagnostic Definitions

The script should compute conservative finite-sample diagnostics from existing data only.

### D1: Monotonicity Proxy

Because existing MC rows are mostly fixed-budget rows, exact monotonicity comparisons S subset T may be sparse. Use two proxies:

1. Direct inclusion pairs if present:

```text
mono_gap(S,T) = G(T) - G(S), for S subset T.
```

2. Search-trace proxy from CD-G:

```text
cd_gain = G_final - G_init.
```

Report:

- number of exact inclusion comparisons;
- fraction with G(T) >= G(S);
- distribution of CD-G improvement.

### D2: Diminishing Returns / Weak Submodularity Proxy

For available inclusion triples A subset B and x not in B:

```text
dr_gap(A,B,x) = [G(A union {x}) - G(A)] - [G(B union {x}) - G(B)].
```

If exact triples are too sparse, report "not identifiable from current artifact" and fall back to:

- B=2 pairwise redundancy from Gate 3a / Gate 3b;
- B=3/4 pairwise-overcomposition failure from Gate 3b.

Do not invent a submodularity estimate if the required comparisons are missing.

### D3: Higher-Order Residual

Use Gate 3b B=3/4 validation rows.

For deployable Q rows:

```text
residual_A = G - A
residual_Q = G - Q_hat
pair_penalty = Q_hat - A
```

Report:

- mean absolute residual under A and best deployable Q;
- whether Q over-penalizes schedules;
- correlation between pair_penalty and residual_A;
- distribution of residual_Q by B and model.

### D4: Local Smoothness Proxy

Use Phase 2b MC rows within each seed and B.

For row pairs with Hamming distance 2 between schedules, i.e. one-swap neighbors:

```text
distance(S,T) = |S symmetric_difference T| / 2
smooth_gap = |G(S) - G(T)|
```

Report by distance bucket:

- one-swap;
- two-swap;
- larger.

Metrics:

- mean |G(S)-G(T)| by distance;
- Spearman correlation between schedule distance and |G difference|;
- whether nearby schedules have meaningfully closer G than random pairs.

### D5: Search Trace Explanation

Compare Phase 3a final schedules to Phase 2b MC schedule pool.

For each B and seed:

- CD-G final G;
- BS-AG final G;
- MC pool quantile of that G for the same seed and B;
- schedule distance from CD-G / BS-AG final schedule to nearest high-MC schedule, if available;
- n_g_calls versus gain.

Report whether search succeeds by:

- finding unusually high-G schedules relative to MC pool;
- exploiting local improvements from initial schedules;
- using many true-G calls to bypass weak surrogate quality.

### D6: Bayesian Optimization Suitability

This is a diagnostic, not an implementation of BO.

A BO direction is plausible if:

- local smoothness is positive;
- schedule distance or phase composition predicts G differences better than random;
- search final schedules occupy coherent regions of schedule space;
- true-G calls are expensive enough that sample efficiency matters.

Write a `bo_suitability` object:

```json
{
  "verdict": "promising" | "unclear" | "not_supported",
  "reasons": ["..."],
  "blocking_gaps": ["..."]
}
```

## Task 3: Implement Core Analysis Script

**Files:**

- [ ] **Step 1: Add CLI, loaders, and pure helper functions**

The script must expose:

```python
def canonical_schedule(steps: list[int] | tuple[int, ...]) -> tuple[int, ...]:
    return tuple(sorted(set(int(s) for s in steps)))


def schedule_distance(a: list[int] | tuple[int, ...], b: list[int] | tuple[int, ...]) -> int:
    sa = set(canonical_schedule(a))
    sb = set(canonical_schedule(b))
    return len(sa.symmetric_difference(sb)) // 2


def is_subset_schedule(a: list[int] | tuple[int, ...], b: list[int] | tuple[int, ...]) -> bool:
    return set(canonical_schedule(a)).issubset(set(canonical_schedule(b)))


def summarize_distribution(values: list[float]) -> dict[str, float | int | None]:
    if not values:
        return {"n": 0, "mean": None, "median": None, "p05": None, "p95": None}
    import numpy as np
    arr = np.asarray(values, dtype=float)
    return {
        "n": int(arr.size),
        "mean": float(arr.mean()),
        "median": float(np.median(arr)),
        "p05": float(np.quantile(arr, 0.05)),
        "p95": float(np.quantile(arr, 0.95)),
    }
```
